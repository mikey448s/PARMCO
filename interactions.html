> This is one of the most classic hardware phenomena in robotics! You have just discovered the battle between Static Friction (Stiction) and Kinetic Friction. It takes significantly more power to start a motor from a dead stop (overcoming static friction) than it does to keep it spinning. Because your code couldn't start the motor at low PWMs, the feedback loop sat there saying: "Still 0 RPM! Add more power! Add more power!" By the time the power was finally high enough to break the stiction (around 10%), the motor violently kicked on. But because it had so much power, it rocketed to 1800 RPM. The feedback loop saw 1800 RPM, panicked, and subtracted a massive chunk of power (e.g., dropping the duty cycle by 9% instantly). That sudden drop starved the motor, causing it to stall back to 0, and the vicious cycle started all over again.
