01 // KERNEL BYPASS [pigpio]

We do not ask the Linux scheduler for permission to toggle pins. Using the pigpio C library, PARMCO utilizes Direct Memory Access (DMA) to seize control of the Broadcom silicon.

The result? A mathematically flawless, hardware-driven 50Hz PWM square wave pumped directly into the L293D H-Bridge. No jitter. No CPU overhead. Total silicon dominance.

gpioSetPWMfrequency(PIN_ENA, 50);
gpioPWM(PIN_ENA, hardware_pwm_val);

03 // THE 1.8V OVERRIDE

The Raspberry PiHat artificially throttled our GPIO logic to a pathetic 1.8V—insufficient to drive the 4N25 optoisolator. Instead of complaining, we engineered a current-sinking bypass.

By routing the 4N25 anode to a constant 3.3V rail and using a 2N3904 NPN transistor on the cathode, a weak 1.7V GPIO signal easily saturates the NPN, snapping the circuit closed and slamming ~11.5V into the IRFZ34N MOSFET gate.

Vbe(sat) = 0.65V
V_GPIO = 1.7V (SUFFICIENT TO SWITCH)

04 // PRE-COGNITIVE FEEDBACK

A pure PID loop must "guess" its way to a target RPM, resulting in wind-up, stiction-stalls, and oscillation. PARMCO is pre-cognitive.

The C-core executes an empirical Quadratic Transfer Function to instantly jump to a calculated baseline duty cycle. Dual HW-201 IR sensor interrupts then feed true-RPM back to the loop. If the motor approaches a stiction stall, an enforced 6.2% minimum duty floor catches it.

duty = (0.00000125 * r * r) - (0.0011 * r) + 6.7;
if (final_duty < 6.2) final_duty = 6.2; // Stall Floor
